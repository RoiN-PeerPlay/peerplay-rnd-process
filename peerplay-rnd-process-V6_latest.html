Task Breakdown

How We Break Down Features

📖 Task Business or Technical Topic One complete idea — the single source of truth
🔧 Dev Sub-Task
🧪 QA Sub-Task
🐛 Bug / Task Bug
🎨 Art / 3rd Party
Rule 1: The Task represents the full idea or feature — it is the single source of truth and is never a subtask of something else.
Rule 2: Sub-tasks define the implementation path — each sub-task describes how we get to the Task goal (Dev builds it, QA validates it).
Rule 3: Each sub-task must be independently deliverable and testable — no sub-task should depend on another being complete first.

Action Plan

Steps to Get There

1
Setup Configure Jira Bug Workflow
New bug statuses: To Do → In Progress → In Review → Ready for QA → Testing → Passed QA → Ready for Release → Close
2
Setup Define Story / Task Structure in Jira
Story/Task as top-level unit • Dev Subtask + QA Subtask • Estimates on subtasks only
3
Setup Define Master Branch as Source of Truth
Master is the single stable base — all feature branches are cut from master and synced with it continuously via fast-forward merges
4
Setup Define Release Branch & Process
Release branch cut from master at code-freeze • Only hotfixes merged in • Tagged on ship • Merged back to master post-release
5
Integrate Set Up CI/CD Pipeline
Automated build & test on every PR • Deploy to Dev on merge to master • Staging & Production gated behind passing checks
6
Integrate TLD Template in Notion
Technical Design Doc template ready for Dev to use from day one
7
Integrate Testomatio + Notion AI Integration
AI auto-generates test cases from PRD & TLD — QA starts with a ready-made test plan
8
Integrate GitHub ↔ Jira PR Linking
Every PR references a Jira task or bug — no orphan code changes
9
Execute Team Alignment Session
Walk Product, Dev & QA through the full new process together
10
Execute First Feature Under New Process
Run end-to-end: PRD → TLD → TDR → Tasks → Dev+QA → Release
11
Improve Retrospective & Lock It In
What worked, what didn't — refine and make the process permanent
Same team. Clearer process. Better games. 🎮
